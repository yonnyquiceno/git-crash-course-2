¿Qué es git

"Un google drive para código"
Git es una herramienta que realiza una función del control de versiones de código de forma distribuida , de la que destacamos varias características:
  • Es muy potente
  • Fue diseñada por Linus Torvalds
  • Es software libre
  • Con ella podemos mantener un historial completo de versiones
  • Es muy rápida

Basic configuration

set global username and email configuration

Open command line (eg git bash)

Set your username: 

git config --global user.name "FIRST_NAME LAST_NAME"

Set your email address: 

git config --global user.email "OUR_NAME@example.com"
Verify your configuration by showing username and email 
# show username 
 git config user.name  
 # show email 
 git config user.email

Manejo de ramas



Git Tiene un sistema de trabajo con ramas que lo hace especialmente potente las ramas están destinadas a provocar versiones divergentes de una version principal, para hacer experimentos o para probar nuevas funcionalidades. Las ramas pueden tener una línea de progreso diferente de la rama principal donde está el core de nuestro desarrollo. En algún momento podemos llegar a probar algunas de esas mejoras o cambios en el código y hacer una fusión a nuestro proyecto principal

Ordenes Básicas

Git tiene muchísimos comandos, pero no hay que aprenderselos todos, a medida que se usan, se memorizan



Iniciar un repositorio vacío en unas carpeta específica.

git init

Clonar un repositorio en el directorio actual

git clone "url del repositorio"

Traer cambios desde el repositorio (actualizar tu rama)

git pull origin "url del repositorio"

Mostrar el estado actual de la rama(branch), como los cambios que hay sin hacer commit.

git status

Listar las ramas existentes

git branch

Crear una nueva rama

git branch "Nombre de la rama"

"pasarse" a la nueva rama

git checkout "Nombre de la rama"

Añadir un archivo especifico.

git add “nombre_de_archivo”

Añadir todos los archivos del directorio

git add .

Quitar de la lista un archivo

git checkout "ruta al archivo"

Confirmar los cambios realizados. El “mensaje” generalmente se usa para asociar al commit una breve descripción de los cambios realizados. 

git commit –m “mensaje”

Subir la rama(branch) “nombre_rama” al servidor remoto.

git push origin “nombre rama”

Git Commit

a commit message shows whether a developer is a good collaborator.

The "commit" command is used to save your changes to the local repository.
Note that you have to explicitly tell Git which changes you want to include in a commit before running the "git commit" command. This means that a file won't be automatically included in the next commit just because it was changed. Instead, you need to use the "git add" command to mark the desired changes for inclusion.
Also note that in Git (not like in Subversion), a commit is not automatically transferred to the remote server. Using the "git commit" command only saves a new commit object in the local Git repository. Exchanging commits has to be performed manually and explicitly (with the "git fetch", "git pull", and "git push" commands).


reference

7 rules for a good commit message

  • Use imperative statements in the subject line, e.g. "Fix broken Javadoc link"
  • Begin the subject line sentence with a capitalized verb, e.g. "Add, Prune, Fix, Introduce, Avoid, etc"
  • Do not end the subject line with a period
  • Keep the subject line to 50 characters or less if possible
  • Wrap lines in the body at 72 characters or less
  • Mention associated jira issue(s) at the end of the commit comment, prefixed with "Issue: " as above
  • In the body of the commit message, explain how things worked before this commit, what has changed, and how things work now

About Pull Requests

About pull requests Pull requests let you tell others about changes you've pushed to a branch in a repository on GitHub. Once a pull request is opened, you can discuss and review the potential changes with collaborators and add follow-up commits before your changes are merged into the base branch.

Otras Ordenes que podrán necesitar

Revertir el commit identificado por "hash_commit"

git revert “hash_commit"

Verificar quién escribió cada linea de código

git blame "ruta al archivo"

“congelar” el estado en el que se encuentra el proyecto en un momento determinado, con todos los cambios que tenemos a "sin comitear", y lo guarda en una pila provisional brindando la posibilidad de poder recuperarlo más adelante. 

git stash

Verificar que hay en stash

git stash show

Volver a traer los cambios luego de hacer stash

git stash apply

Ver historial de commits

git log

Ver el árbol git

git log --graph --oneline --all
